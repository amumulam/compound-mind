## Git Hook 检查机制详解

**时间**：00:39

**机制**：
- `.git/hooks/pre-commit` 在 commit 前自动执行
- 检查目录结构是否符合规范
- 如果不符合，输出错误并阻止提交

**优点**：自动化、强制执行、及时反馈
**缺点**：事后检查、可绕过、只对 git 操作生效


---

## Git Hook 方案的致命局限

**时间**：00:41

**问题**：如果 agent 不使用 git，Git Hook 永远不会触发

**约束力层级**：
| 层级 | 机制 | 强度 |
|------|------|------|
| 声明式 | 文档定义规范 | 最弱 |
| 检查式 | Git Hook / Heartbeat | 中等 |
| 强制式 | 工具层面拦截 | 最强 |

**框架局限性**：
- 框架只能"声明"和"检查"
- 无法"强制执行"
- 真正的强制执行需要 OpenClaw 核心支持

## 框架优化方案：多层防护机制

**时间**：00:44

**方案**：三层防护
1. **事前预防**：Session 启动时强制读取目录规范
2. **事中检查**：Heartbeat 定期扫描目录结构
3. **事后拦截**：Git Hook 提交前检查

**效果**：即使某一层失效，其他层仍可发现问题
