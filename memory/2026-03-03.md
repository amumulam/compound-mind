### bun 代理问题排查

**问题**：bun 下载超时，curl 报 `Failure when receiving data from the peer`

**原因**：mihomo 代理在运行（端口 7890），但环境变量没有设置，bun 安装脚本内部的 curl 没有走代理

**解决**：
```bash
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890
bun x @every-env/compound-plugin install compound-engineering --to openclaw
```

**注意**：`bunx` 命令在新版本 bun 中应该用 `bun x` 代替
