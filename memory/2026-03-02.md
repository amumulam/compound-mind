## 配置学习

- `agents_list` 返回的是当前 agent 有权限访问的其他 agent 列表
- 每个 agent 有独立的 `subagents.allowAgents` 配置
- `tools.agentToAgent.allow` 是全局的 agent-to-agent 通信 allowlist
- 修改配置用 `gateway action=config.patch raw={"...": "..."}`

## 知识冲突处理机制（重要！）

**主人教我的方法**：
当发现知识/规则冲突时：
1. 识别出这是冲突（需要积累识别能力）
2. 抛给主人决策
3. 主人决定哪样对、哪样错
4. 更新记忆 + 记录决策理由

**前提条件**：我要先能识别出冲突

**复杂情况**：
- 语境变化导致的"伪冲突"（时间不同，规则不同）
- 条件性规则（表面冲突，实际是不同条件）
- 隐含的深层冲突（表面不冲突，实际执行时矛盾）
- 优先级冲突（同一场景，多个规则适用）

**完整流程**：
```
积累经验 → 提高识别能力 → 发现潜在冲突 → 抛给主人 → 决策 → 更新记忆 + 记录理由
```

**关键点**：记录决策理由，这样下次遇到类似情况就知道"为什么主人选了这个"
