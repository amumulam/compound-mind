## 框架优化

### [[2026-03-07-directory-enforcement]] - 目录规范强制执行机制

**关键词**：目录规范、多层防护、强制执行

**问题**：框架定义了目录规范，但没有机制确保执行，导致文件放错位置

**解决方案**：
- 事前预防：在 AGENTS.md 中添加目录规范表格
- 事中检查：在 HEARTBEAT.md 中添加目录结构检测规则
- 事后拦截：使用标记机制保护用户内容

**学到的经验**：
- 多层防护优于单层防护
- 标记机制的价值
- 约束力层级理论

**相关**：Compound Mind v1.3.0

---

### [[2026-03-07-git-hook-limitations]] - Git Hook 的致命局限

**关键词**：Git Hook、工具局限、多层防护

**问题**：Git Hook 只对 git 操作生效，不使用 git 就不会触发检查

**解决方案**：
- 不依赖单一的 Git Hook
- 采用多层防护机制
- 心跳检查独立于 git

**学到的经验**：
- 理解工具本质能力边界
- 单一防护的脆弱性
- 约束力层级理论

### [[2026-03-05-reuse-over-reinvent]] - 依赖已有方案，不要重复造轮子

**关键词**：代码复用、依赖管理、工程原则

**问题**：倾向于自己设计方案，而不是使用已有的成熟方案

**解决方案**：
- 先检查依赖是否已有解决方案
- 优先使用成熟方案
- 只在必要时才自定义

**学到的经验**：
- "不要重新造轮子"是核心工程原则
- "不要添加没有意义的工作量"
- 成熟方案更可靠、更可维护

## 使用说明

### 如何创建新方案

1. 在 `docs/solutions/` 目录下创建新文件，命名格式：`YYYY-MM-DD-identifier.md`
2. 使用 `TEMPLATE.md` 作为模板
3. 填写完整的问题描述、根因分析、解决方案、学到的经验
4. 在本 INDEX.md 中添加索引条目

### 如何查找方案

1. 根据关键词搜索
2. 按目录分类浏览
3. 查看学到的经验部分，找到类似的场景

### 如何维护索引

每次添加新方案时，同步更新本 INDEX.md：
- 添加到对应的分类
- 提供关键词、问题、解决方案摘要
- 记录学到的经验
- 在更新历史中记录添加日期
