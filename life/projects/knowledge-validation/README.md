# Knowledge Validation Module

## 功能

检测知识的有效性，保持记忆系统的准确性。

## 触发方式

每周日运行，与 pattern extraction 配合。

## 检查规则

### 过时内容检测
- 技术知识：框架版本、API 变化
- 配置信息：endpoint、token 更新
- 状态信息：已完成/已失效的任务

标记方式：在知识条目后加 `[已过时: YYYY-MM-DD]`

### 孤立内容检测
- 没有被引用的知识
- 没有关联的决策
- 超过 30 天未使用的信息

处理方式：归档到 `life/archives/` 或删除

### 重复内容检测
- 相似的知识条目
- 同一决策的多处记录

处理方式：合并，保留最完整的版本

### 冲突检测
- 矛盾的知识条目
- 不一致的决策

处理方式：标记冲突，提交主人决策

## 输出格式

```
## 知识验证报告 YYYY-MM-DD

### 过时内容 (N 条)
- [内容]: 原因

### 孤立内容 (N 条)
- [内容]: 最后使用时间

### 重复内容 (N 条)
- [内容]: 合并建议

### 冲突内容 (N 条)
- [冲突描述]: 需要决策

### 建议
- [清理建议]
```
