# 决策日志模块设计

## 功能

自动识别和记录重要决策，保持可追溯性。

## 决策识别关键词

### 强触发词（必须记录）
- "决定"、"选择了"、"选定"
- "主人说"、"主人确认"、"主人要求"
- "最终方案"、"确定了"

### 弱触发词（可能需要记录）
- "倾向"、"倾向于"
- "考虑"、"考虑到"
- "因为...所以"

## 自动触发机制

### 检查点提取时

当检测到决策关键词时：
1. 提取决策内容
2. 询问主人是否需要记录正式决策
3. 如需要，按格式创建决策文件

### 手动触发

主人说"记录决策"或"记下来这个决定"

## 决策文件模板

见 `life/decisions/README.md`

## 决策验证

定期（每周知识验证时）检查：
- 决策是否仍然有效？
- 前提条件是否变化？
- 是否需要重新决策？

## 与知识验证的关联

当发现新旧决策冲突时：
1. 标记冲突
2. 分析前提条件变化
3. 提交主人重新决策
